# MCN Script Assistant

## 适用场景

- 品牌方向 MCN 下发商单 brief，需要为指定博主生成种草脚本
- 需要确保脚本符合博主原生风格，避免硬广
- 需要自动化质检 + 飞书文档交付

## 适用平台

- 小红书短视频（主要）
- 可适配抖音、B站等短视频平台

## 输入材料

| 材料 | 必填 | 格式 | 说明 |
|------|------|------|------|
| 客户 Brief | ✅ | 文本 | 品牌、产品、卖点、人群、平台要求、禁忌 |
| 博主内容样本 | ✅ | 文本/链接 | 目标博主近期 2-3 条代表作品的内容描述 |
| 博主风格分析 | ✅ | 结构化文本 | 可用 Step 2 Agent 自动生成 |
| 飞书文档 ID | ✅ | 字符串 | 目标飞书文档的 docToken |
| 飞书 API 凭证 | ✅ | App ID + Secret | 用于自动写入文档 |

## 执行步骤

### Step 1: Brief 拆解
**输入**：客户原始 brief
**处理**：使用 `prompts/01-brief-deconstruct.md` 中的 System Prompt，调用 LLM 将 brief 拆解为结构化创作简报
**输出**：创作简报（含卖点映射、禁忌清单、博主适配建议）
**耗时**：~5s

### Step 2: 内容风格分析
**输入**：博主代表作品 + 内容方向描述
**处理**：使用 `prompts/02-style-analysis.md` 中的 System Prompt，提取博主风格指纹
**输出**：风格指纹（钩子模式、语速节奏、镜头语言、口播特征、植入方式）
**耗时**：~5s

### Step 3: 脚本 + 分镜生成
**输入**：创作简报 + 风格指纹 + 目标时长
**处理**：使用 `prompts/03-script-generate.md` 中的 System Prompt，生成脚本
**输出**：完整脚本（含钩子/口播/植入点/CTA）+ 分镜表 + 合规初筛
**耗时**：~10s

### Step 4: 合规质检
**输入**：生成脚本 + 合规红线清单
**处理**：
  1. 确定性规则扫描（硬编码禁用词列表，无需 LLM）
  2. 上下文语义检查（调用 LLM，使用 `prompts/04-compliance-check.md`）
**输出**：质检报告（逐项检查 + 风险等级 + 修改建议）
**判定**：通过 → 进入 Step 5；不通过 → 标记风险项，人工干预或自动重新生成
**耗时**：~5s

### Step 5: 写入飞书文档
**输入**：通过质检的脚本 + 分镜表 + 质检报告
**处理**：使用 `prompts/05-feishu-write.md` 格式化内容 → 调用飞书 API 写入
**输出**：飞书文档链接
**耗时**：~3s

## 输出格式

```
output/
├── final-script.md          # 完整脚本（口播文案+画面描述+字幕）
├── storyboard.md             # 分镜表
└── compliance-report.md      # 合规质检报告
```

飞书文档中同时包含上述三部分内容，排版呈现。

## 风险检查清单

### 一级禁用词（脚本绝对不能出现）
- [ ] 减肥、瘦身、燃脂、掉秤、减脂、刮油、排油
- [ ] 降糖、降血糖、降血压、降血脂
- [ ] 治疗、治愈、疗效、药效、功效
- [ ] 最好、第一、唯一、顶级、最强、首选

### 二级谨慎词（需评估上下文）
- [ ] 代餐 — 仅在"早餐替代"场景下可用，不得暗示替代正餐减肥
- [ ] 低卡 — 建议替换为"轻负担"
- [ ] 控糖 — 不得出现，涉及医疗暗示

### 品牌要求
- [ ] 产品名称拼写正确（轻醒）
- [ ] 卖点表述准确（高蛋白、饱腹、0蔗糖、轻负担）
- [ ] 未超出 brief 授权范围（不自行添加功能/功效）

### 平台合规
- [ ] 商业合作内容已标注
- [ ] 无外链/二维码
- [ ] 无竞品/他品牌提及

### 内容质量
- [ ] 口播读起来像是博主自己说的话（非广告腔）
- [ ] 产品植入自然（非硬推）
- [ ] 钩子有吸引力
- [ ] CTA 自然引导互动

## 技术实现

### 依赖
- Python 3.9+
- `requests`（HTTP 调用，LLM API + 飞书 API）
- DeepSeek API Key（环境变量 `DEEPSEEK_API_KEY` 或 `--api-key` 参数）

### 运行方式

```bash
# 1. 安装依赖
pip install requests

# 2. 设置环境变量
export DEEPSEEK_API_KEY="your-api-key"

# 3. 配置飞书凭证（编辑 feishu/config.json）
# {
#   "app_id": "cli_xxx",
#   "app_secret": "xxx",
#   "doc_token": "YoWEw3GHvio5kUkZdjPc6aY3nCc"
# }

# 4. 运行脚本
python main.py --brief brief.md --blogger references/blogger-profiles.md
```
