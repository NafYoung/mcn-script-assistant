## 一、固定结构公式

### 开头模式（二选一）
- **招呼型**：「Hello 大家，今天分享一个...」
- **痛点引入型**：「如果你也想把早餐吃得健康一点，而且每天不重复，那这个视频一定会帮到你。大家好，我是爱吃早餐的027。」

### 钩子策略
- **结果前置**：「这三种低卡酸奶酱真的太好吃了，拌沙拉、蘸面包都可以，强烈推荐。」
- **痛点共鸣**：「如果你也想把早餐吃得健康一点...」
- 钩子后立刻切入正题，不铺垫

### 结尾（固定句式，不可修改）
**「我是零二七/027，我们一起好好吃饭。」**

### 过渡与情绪词库
| 功能 | 她的原话 |
|------|---------|
| 引出重点 | "重点来了" |
| 强调推荐 | "强烈推荐""真的超级推荐""简直完美" |
| 表达好吃 | "真的好好吃啊""很好吃""太好吃了" |
| 鼓励尝试 | "赶紧试试吧""一定要试试" |
| 段落收尾 | "好啦，今天就分享到这里" |

## 二、口播语言特征

### 句式特点
- **短句为主**，平均句长 15-20 字
- 大量使用"真的""超级""特别"等口语强调词
- 极少用连接词，靠语义自然流转
- 问句极少，几乎全是陈述+推荐

### 人称使用
- 第一人称「我」贯穿始终
- 对观众用「你」：直接、亲近
- "我们"仅出现在结尾固定句式：「我们一起好好吃饭」

### 产品/食材介绍方式（⭐ 最关键）
**她从不在口播中念品牌名。** 介绍食材的方式是描述属性：
- 「一定要是这种比较浓稠的」
- 「希腊酸奶一定要选无糖的」
- 「我用的是这个 Skippy，无糖有颗粒感的」（花生酱破例提了一次牌子）

→ 商单脚本中，「轻醒」不可出现在口播文案中。通过画面展示瓶身标签。

## 三、视频结构模式

### 短食谱型（~60s）
```
招呼(3s) → 食材展示(10s) → 做法①(15s) → 做法②(15s) → 做法③(10s) → 推荐结尾(7s)
```

### 多品合集型（~90s）
```
钩子(5s) → 单品①(25s) → 单品②(25s) → 单品③(25s) → 署名结尾(5s)
```
每个单品的 sub-structure：**是什么 → 怎么做 → 怎么用 → 为什么好**

### 一周备餐型（~120s）
```
痛点引入(10s) → 方法论(10s) → 单品①-⑤(各15s) → 总结公式(15s) → 署名结尾(5s)
```

## 四、品牌匹配的天然入口

凌二七的三大内容习惯与「轻醒」酸奶完美对接：

| 她的习惯 | 产品植入方式 |
|---------|------------|
| 反复强调"希腊酸奶一定要选无糖浓稠的" | 镜头切到轻醒瓶身，标签"0蔗糖""高蛋白"自然露出 |
| 以酸奶为基底做酱/蘸料 | 制作过程中用轻醒替代无品牌酸奶 |
| "高蛋白""低负担""不油腻"是她高频措辞 | 这些词本身就是轻醒的卖点，口播不改，画面配合 |

### 一句话植入策略
**口播保持她的语言习惯（不念品牌），画面让产品说话。**

## 五、合规对照

### 她在自己的内容中使用的禁用词（我们脚本中必须避开）

| 她说的 | 出现视频 | 我们的处理 |
|--------|---------|-----------|
| "减脂零食""减脂期""想减脂的话" | 视频1、2 | 直接删除，不找替代词 |
| "低卡" | 视频2 tag | 替换为"轻负担"或不提 |
| "控糖期" | 视频3 | 直接删除 |
| "热量不高" | 视频2 | 替换为"吃起来没负担" |

### 她已在用的安全措辞（可直接复用）
- "低负担""干净饮食""健康饮食" — 这是她原生词汇，安全
- "能吃饱但又不油腻""不会那么噎人" — 体验描述，非功效承诺
- "无糖""浓稠""高蛋白" — 产品客观属性，安全
